## 📥 下载与安装

### 获取程序

1. 打开 [GitHub Releases 页面](https://github.com/eghrhegpe/ysm-model-manager/releases)
2. 找到最新的版本（如 `v1.4.3`）
3. 下载 `YSM-Model-Manager_windows_amd64.zip`
4. 解压到任意目录（建议 `D:\YSM-Model-Manager\` 或其它非系统盘位置）

### 首次启动

双击 `YSM-Model-Manager.exe` 启动程序。首次启动会弹出设置页面，需要配置两个路径才能正常使用：

---

## ⚙️ 首次配置

### 1. 游戏根目录（.minecraft 路径）

点击「游戏根目录」旁的 📂 按钮，选择你的 `.minecraft` 文件夹。

**常见位置：**

| 启动器     | 默认路径                         |
| ---------- | -------------------------------- |
| PCL2       | `C:\PCL2\.minecraft`             |
| HMCL       | `C:\Users\你的用户名\.minecraft` |
| 官方启动器 | `C:\Users\你的用户名\.minecraft` |
| BakaXL     | `C:\Users\你的用户名\.minecraft` |

程序会自动扫描 `versions`、`assets`、`launcher_profiles.json` 等特征文件来验证路径是否正确。如果选择的是父目录，程序会尝试自动修正为 `.minecraft` 子目录。

> 💡 也可以点击「🔍 自动搜索」让程序扫描 10 多个常见位置。

### 2. 模型仓库路径

点击「模型仓库路径」旁的 📂 按钮，选择一个**空文件夹**或**已有模型文件的文件夹**。

这个目录是用来存放你所有 YSM 模型文件的地方。程序会递归扫描该目录下的 `.ysm`、`.zip`、`.7z` 文件。

> ⚠️ 建议仓库路径和游戏根目录在**同一分区**，否则硬链接模式不可用（会降级为复制）。

### 3. 链接模式

三种安装方式，在设置页的下拉菜单中切换：

| 模式         | 图标 | 推荐？      | 原理                                             | 要求                       |
| ------------ | ---- | ----------- | ------------------------------------------------ | -------------------------- |
| **硬链接**   | 🔗   | ✅ **推荐** | 仓库和整合包共用磁盘数据，不占额外空间，同步即时 | 仓库和整合包必须在同一分区 |
| **复制**     | 📋   | ⚠️ 备选     | 文件复制到整合包目录，占用双倍空间               | 无限制                     |
| **符号链接** | 🔗   | ❌ 不推荐   | 类似快捷方式但更底层                             | YSM 模组不兼容             |

**硬链接是最佳选择**：仓库启用/禁用模型时，整合包会立即同步，无需手动操作。如果跨分区，程序会自动降级为复制模式。

## 🧭 功能导航

左侧导航栏共 7 个页面：

### 📦 模型仓库

浏览和管理你的模型文件集合。

- **树形浏览**：左侧文件树按文件夹结构展示所有模型
- **搜索框**：按文件名搜索，匹配字符高亮
- **排序**：可按名称、大小、修改日期排序
- **启用/禁用**：点击文件前面的开关切换（禁用后文件加 `.ban` 后缀，在游戏中不显示）
- **复选框批量操作**：勾选多个文件后批量启用/禁用
- **文件夹开关**：点击文件夹旁边的按钮，批量开关该文件夹下所有模型（绿色=全启用，灰色=全禁用，黄色=混合）
- **右键菜单**：禁用/启用、查看模型详情、打开所在文件夹
- **3D 预览**：点击模型文件，右侧面板显示 3D 骨骼模型（支持旋转、缩放）
- **拖拽移动**：在文件树中拖拽文件到不同文件夹

### 🎮 整合包管理

管理你的 Minecraft 整合包中的 YSM 模型。

- 自动检测所有已安装的 Minecraft 版本实例
- 每个整合包显示其 YSM 模组状态（🚫 无YSM = 该整合包没有安装 YSM 模组）
- **同步状态四象限**：

| 状态          | 含义                   | 操作           |
| ------------- | ---------------------- | -------------- |
| ✅ 已同步     | 仓库有且已安装到整合包 | —              |
| ⬇️ 待同步     | 仓库有但未安装         | 点击「安装」   |
| ⚠️ 已禁用     | 仓库已禁用但之前已安装 | 仓库启用即可   |
| 📤 可加入仓库 | 整合包有但仓库无       | 点击上传到仓库 |

- **搜索筛选**：支持按文件名搜索和「仅显示有 YSM 的整合包」
- **卡片折叠**：点击整合包标题展开/折叠，状态持久化保存

### 🎨 创作者频道

浏览 YSM 模型创作者列表和在线资源。

- **站点管理**：B站、爱发电等平台的快速跳转
- **预设搜索**：一键搜索「YSM 免费模型」「YSM 模型分享」等
- **创作者数据库**：87+ 位 YSM 创作者列表，按平台分类
- **内嵌浏览器**：在应用内直接浏览网页（无需切换窗口）

### 🧩 创意工坊

在线浏览和下载 GitHub 上的 YSM 模型仓库。

- **仓库列表**：已配置的 GitHub 仓库源
- **模型浏览**：读取远程 `index.json` 在线浏览模型列表
- **一键下载**：点击 ⬇️ 直接下载到本地仓库

### 👴 仓库元老

在「模型仓库」页面点击「👴 仓库元老」标签查看。

按文件修改日期排序，展示仓库中最早 4 个模型及其健康度和月度活动。

- **健康度**：综合禁用比例和重复文件评分的仓库健康指示
- **月度活动**：12 个月格子显示各月模型活动量
- **今日推荐**：随机 3 个模型推荐，点击可查看详情

### 🛠️ 诊断与冲突

- **操作日志**：所有安装、导入、删除操作的历史记录，失败原因分行显示
- **模型去重**：按 SHA256 检测仓库中的重复文件，逐组选择保留哪个副本（其余移入回收站）
- **冲突检测**：扫描同名文件存在于多个整合包中的情况

### ⚙️ 设置

- **游戏根目录**：配置/自动搜索 `.minecraft` 路径
- **模型仓库路径**：配置模型文件存放目录
- **链接模式**：切换安装方式（见上）
- **主题模式**：4 套主题可选
  - 💻 跟随系统
  - 🌙 赛博霓虹（深色）
  - ☀️ 温暖木纹（浅色）
  - ⚪ 极简深邃
- **回收站**：查看和恢复已删除的模型文件
- 导航状态和窗口位置会在关闭时自动保存

## 🔐 安全机制

- **文件校验**：导入时验证魔数（ZIP/7z），文件大小上限 500MB
- **路径防护**：所有文件操作都经过路径遍历攻击防护
- **回收站**：删除的文件移入 `.recycle` 目录，可恢复
- **硬链接/符号链接识别**：回收站对硬链接和符号链接采用不同的删除策略（直接删除，不影响原始数据）
- **shutdown 保护**：关闭时自动恢复可能的 panic

## ❓ 常见问题 (FAQ)

### Q: 下载了 exe 但打不开 / 闪退？

**可能原因：**

1. **缺少 WebView2 运行时** — Wails 应用需要 WebView2。Windows 10/11 通常已预装，如果缺失，请从 [Microsoft 官网](https://developer.microsoft.com/en-us/microsoft-edge/webview2/) 下载安装。
2. **杀毒软件拦截** — 部分杀毒软件会误报 Wails 应用。请将程序目录加入白名单。
3. **路径含中文/空格** — 请确保程序所在目录路径不含特殊字符。

### Q: 三种链接模式怎么选？

- **如果你只玩一个整合包，仓库和 .minecraft 在同一分区** → 选**硬链接**（最省空间，同步最快）
- **如果跨分区** → 程序自动降级为**复制**，无需手动切换
- **不要选符号链接** — YSM 模组不识别符号链接

### Q: 模型启用/禁用了，但游戏中没变化？

1. 确保整合包中安装了 YSM 模组（版本 1.0.0+）
2. 尝试按 F3+T 重载资源包
3. 如果使用硬链接模式，禁用操作是**即时同步**的；复制模式需要手动重新安装

### Q: 导入模型时提示「未检测到模型文件」？

1. 确认文件后缀是 `.ysm`、`.zip` 或 `.7z`
2. 如果是 `.zip` 文件，确保内部包含 `minecraft:geometry` 格式的 JSON
3. 可以尝试勾选「跳过文件头校验」（仅当你确定文件来源安全时）

### Q: 如何添加新的创作者到数据库？

创作者信息存储在仓库根目录的 `creators.json` 中。可以手动编辑该文件，或在「创作者频道」使用导入功能。格式示例：

```json
{
  "name": "创作者名称",
  "desc": "简介",
  "type": "bilibili"
}
```

`type` 可选值：`bilibili`、`afdian`、`github`，也可以用分号组合如 `bilibili;afdian`。

### Q: 如何生成 GitHub 索引供在线浏览？

在「模型仓库」页面，点击「📇 生成索引」按钮。程序会扫描整个仓库并生成 `docs/index.json`。将此文件提交到 GitHub 仓库，即可在「创意工坊」中在线浏览。

### Q: 如何更新程序？

启动时程序会自动检查 GitHub Releases 的新版本。如果发现新版本，右下角会弹出提示，点击即可下载并自动更新（无黑框，完成后自动重启）。

也可以手动从 GitHub Releases 页面下载最新 ZIP，解压覆盖即可。

### Q: 日志在哪里？

程序会在桌面生成 `ysm-debug.log` 文件，记录运行时的调试信息。如果遇到问题，可以查看此日志文件。

### Q: 如何备份我的配置？

配置保存在 `ysm_config.json` 文件中（与 exe 同目录）。备份此文件即可保留所有设置。模型仓库本身是你的模型文件集合，建议另外备份。

### Q: 回收站里的文件怎么恢复？

在「设置」页面底部有「回收站」区域，可以查看、恢复或永久删除已回收的文件。恢复的文件会回到原位置。

## 🐛 故障排查

| 症状                   | 检查项                                                  |
| ---------------------- | ------------------------------------------------------- |
| 启动后页面空白         | 检查 WebView2 是否安装；尝试以管理员身份运行            |
| 模型列表为空           | 确认仓库路径配置正确，目录下有 `.ysm`/`.zip`/`.7z` 文件 |
| 按钮点击无反应         | 尝试重启程序；如果刚更新过，确保旧进程已完全退出        |
| 整合包列表为空         | 确认游戏根目录配置正确，目录下有 `versions` 文件夹      |
| 「安装」按钮卡在导入中 | 检查整合包 `custom` 目录是否被游戏锁定（关掉游戏重试）  |
| 3D 预览不显示          | 部分高面数模型（>2000 cubes）可能卡顿，稍等片刻         |
