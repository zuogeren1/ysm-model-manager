# PCL2 功能对比分析

> 目标：明确 YSM Model Manager 的独特价值，避免与 PCL2 功能重叠。
> 分析基于 PCL2 v2.12.8.2（2026-05-24 发布），VB.NET 实现。

---

## 一、PCL2 已完善的通用功能

这些功能 PCL2 已经做得很好，YSM Model Manager **不需要重复造轮子**：

### 1.1 Minecraft 启动 & 版本管理

- [x] 多版本共存（正式版/快照版/远古版）
- [x] 加载器安装（Fabric / Forge / OptiFine / Quilt / LiteLoader）
- [x] 版本隔离（每个版本独立 mods/config 目录）
- [x] Java 自动检测 & 参数配置
- [x] 游戏内存/分辨率/JVM 参数

### 1.2 模组管理

- [x] 模组下载（集成 CurseForge / Modrinth / MC百科）
- [x] 模组启用/禁用
- [x] 模组依赖检测
- [x] 模组冲突提示

### 1.3 整合包管理

- [x] CurseForge / Modrinth 整合包一键安装
- [x] 整合包导出/分享

### 1.4 资源包/光影包管理

- [x] 下载（CurseForge / Modrinth）
- [x] 列表浏览 + 启用/禁用
- [x] 拖拽安装
- [x] PCL2 的资源包管理功能**远超我们刚做的 `<app-resource-manager>`**

### 1.5 世界/存档管理

- [x] 世界列表 + 备份 + 删除
- [x] 存档导入/导出

### 1.6 账户 & 联机

- [x] 微软账户登录
- [x] 离线账户
- [x] 简洁联机（内网穿透）

## 三、功能重叠区域 & 策略

### 3.1 材质包/光影包管理

**现状**：我们刚做了一个轻量版（列表 + 导入 + 启用/禁用 + 删除）。
**PCL2**：下载源集成、拖拽安装、排序、详细元数据展示。

**建议**：**到此为止，不再深入**。用户若有高级需求（下载、排序），应该用 PCL2。我们的材质包 tab 定位是"快速开关当前 MC 实例的材质包"，不是"完整的材质包管理器"。

### 3.2 MC 版本管理

**现状**：我们有版本列表 + 实例选择。
**PCL2**：下载、安装加载器、版本隔离、配置启动参数。

**建议**：**保持现有级别**。版本列表只做浏览和选择，下载/安装/加载器全交给 PCL2。

### 3.3 实例管理

**现状**：我们以"版本"为核心，每个版本下面装 YSM 模型。
**PCL2**：以"实例"为核心，每个实例独立 mods/config/资源包。

**建议**：**保持差异化**。我们的"实例"核心是装 YSM 模型，不是完整的 MC 实例管理。如果用户需要完整的实例管理，可以用 PCL2。

## 四、发现的新赛道：模组通用资产

PCL2 管的是**模组本身**（安装/版本/依赖），而模组产生的**通用资产**（YSM 模型、机械动力蓝图等）是 PCL2 完全没覆盖的领域。

### 4.1 什么是"模组通用资产"？

| 资产类型         | 来源模组     | 文件格式              | 共享属性             |
| ---------------- | ------------ | --------------------- | -------------------- |
| **YSM 模型**     | YSM / Figura | `.ysm`                | 跨版本通用，玩家创作 |
| **机械动力蓝图** | Create Mod   | `.nbt` / `.schematic` | 跨版本通用，玩家创作 |
| **建筑模板**     | Litematica   | `.litematic`          | 跨版本通用           |
| **红石图**       | 原版 + 投影  | `.nbt`                | 可分享               |

这些资产的共同特点：

- **不依赖模组版本**（蓝图的方块数据跨 MC 版本基本兼容）
- **玩家创作产物**（不是从 CurseForge 下载的，是社区分享的）
- **需要管理**（浏览、预览、搜索、归类、备份）
- **PCL2 完全不支持**（它只管理 mods/config/resourcepacks）

### 4.2 我们的差异化优势

```
PCL2（模组层）                    YSM Model Manager（资产层）
┌─────────────────┐              ┌──────────────────────────┐
│ 安装 Create 模组 │              │ 管理 Create 蓝图          │
│ 更新 Create 版本 │              │ 预览蓝图结构               │
│ 调试模组冲突     │              │ 按建筑类型分类/搜索         │
│ 开关资源包       │              │ 跨实例同步蓝图              │
│ 下载整合包       │              │ 社区分享蓝图                │
└─────────────────┘              └──────────────────────────┘
       ↓                                  ↓
  启动 Minecraft                       管理 Minecrart 创作
```

### 4.3 架构调整思路

现在我们有：

```
resource_types.json
├── ysm            → 文件树 + 3D 预览 + 治理      🔴 核心
├── resourcepack   → 轻量列表 + 导入/切换          ⚪ 辅助
└── shaderpack     → 轻量列表 + 导入/切换          ⚪ 辅助
```

可以扩展为：

```
resource_types.json
├── ysm            → 文件树 + 3D 预览 + 治理      🔴 核心
├── resourcepack   → 轻量列表 + 导入/切换          ⚪ 辅助
├── shaderpack     → 轻量列表 + 导入/切换          ⚪ 辅助
├── create-blueprint → 列表 + 预览 + 导入/导出     🟡 新赛道
├── litematic      → 列表 + 预览 + 导入/导出       🟡 待评估
└── ...            → 按需扩展
```

每种资产类型都可以复用 `<app-resource-manager>` 组件，只需配置不同的：

- 文件格式（`.nbt` / `.schematic` / `.litematic`）
- 预览方式（3D 结构预览 / 缩略图 / 无预览）
- 导入策略（`SimpleCopyImporter` 或自定义）
- 存放目录（`schematics/` / `litematics/` 等）

## 五、产品定位（修订版）

```
不是"又一个 Minecraft 启动器"
也不是"YSM 模型专属管理工具"
而是 "Minecraft 模组通用资产管理器"
```

| 层级       | 工具                  | 职责                       |
| ---------- | --------------------- | -------------------------- |
| **启动层** | PCL2 / 官方启动器     | 启动游戏、管理模组、整合包 |
| **资产层** | **YSM Model Manager** | 管理模组产生的通用资产     |

### 支持的资产类型

| 类型            | 状态        | 优先级    |
| --------------- | ----------- | --------- |
| YSM 模型        | ✅ 已支持   | 🔴 核心   |
| 材质包/光影包   | ✅ 轻量辅助 | ⚪ 保持   |
| 机械动力蓝图    | 🟡 新赛道   | 🟡 待开发 |
| Litematica 投影 | 🟡 新赛道   | ⚪ 待评估 |

### 功能优先级

1. 🔴 **YSM 模型管理** — 核心，持续投入
2. 🟡 **创意工坊/模型下载** — YSM 生态扩展
3. 🟡 **机械动力蓝图管理** — 第二个资产类型验证
4. 🟢 **整合包安装模型** — 差异化场景
5. ⚪ **材质包/光影包切换** — 辅助功能，维持轻量
6. ❌ **MC 启动/模组安装/整合包下载** — 交给 PCL2

4. ⚪ **材质包切换** — 辅助功能，维持轻量
5. ⚪ **光影包切换** — 辅助功能，维持轻量
6. ❌ **MC 启动** — 不做，交给 PCL2
7. ❌ **模组管理** — 不做，交给 PCL2
8. ❌ **下载资源包/光影包** — 不做，交给 PCL2
