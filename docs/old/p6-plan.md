## 1️⃣ WASM 能力补齐（解同版本 .ysm）✅

| 项目     | 内容                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| **目标** | 让嵌入式 WASM 能解码当前 CLI 支持的所有 .ysm 版本，彻底消除 exe sidecar 依赖 |
| **难度** | ⭐⭐⭐⭐ 极高                                                                |
| **状态** | ✅ 完成（2026-06-08）                                                        |

### 最终方案

不是编译参数问题，而是数据传递链路（Go base64 → JS → WASM HEAPU8）损坏。详见
`docs/postmortem-20250608-wasm.md`。

---

## 2️⃣ 双解码去重（缓存 YSMParser 输出）✅

| 项目     | 内容                                                                 |
| -------- | -------------------------------------------------------------------- |
| **目标** | 当前预览纹理 + 预览骨骼各跑一次 YSMParser，合并为一次解码 + 缓存复用 |
| **难度** | ⭐⭐ 中等                                                            |
| **状态** | ✅ 完成（2026-06-08）                                                |

### 实现

- 新增 `_previewDataCache = new Map()`（全局路径级缓存）
- `_loadPreviewImage` 和 `_loadModel2D` 都优先查缓存
- Go 端 `AnalyzeBedrockModel` 一次性返回 `{texture, geometry}`，存入缓存
- WASM 路径 `_decodeYsmViaWasm` 返回 `{ texture, geometry }`，`_ysmCache` 实例级缓存

## 8️⃣ MC 目录智能检测 ✅

| 项目     | 内容                                                           |
| -------- | -------------------------------------------------------------- |
| **目标** | 自动发现 .minecraft 目录，用户选错时自动修正，提升首次使用体验 |
| **难度** | ⭐⭐ 中等                                                      |
| **状态** | ✅ 完成（2026-06-08）                                          |

### 实现

**Go 端 (`app.go`)**:
- `isLikelyMinecraftDir()` — 内容级检测：检查 `versions/`、`assets/`、`launcher_profiles.json`，不再仅靠路径名
- `scanMinecraftDirs()` — 扫描 10+ 个常见位置（PCL2/PCL/MC/Minecraft/Games 等各盘符）
- `GetMinecraftPaths()` — 返回所有检测到的 .minecraft 目录（Wails Binding）
- `ValidateMinecraftDir(dir)` — 验证用户选择的目录：直接检查 → 自动补 `.minecraft` 子目录 → 检查 `versions/`，返回修正后的路径或错误提示
- `SaveAppConfig` 中集成了自动修正

**前端 (`app-sidebar/events.js`)**:
- 首次启动自动调用 `GetMinecraftPaths()`，取第一个有效路径设为默认
- MC 目录按钮自动显示检测到的路径

**前端 (`workshop-settings.js`)**:
- "自动搜索"按钮改用 `GetMinecraftPaths()`，不再解析字符串

**前端 (`app-sidebar/index.js`)**:
- `dir:select-mc` handler 捕获 `SaveAppConfig` 错误并 toast 提示

### 用户选父目录的自动修正

选了 `D:/PCL2/` → `ValidateMinecraftDir` 自动检查 `D:/PCL2/.minecraft/` → 存在则自动跳转
