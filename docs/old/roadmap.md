# 路线图

## P0 — 下一版本必须

> ✅ 全部 P0 已完成

### 1. 整合包模型去重

- **现状：** ✅ **已实现** — 右键整合包 → "🔁 去重此整合包模型"
- **技术：** Go 端新增 `DeduplicateCustomDir(customDir string) (int, int, error)`，扫描 custom 目录按 SHA256 哈希分组，每组保留第一个，其余调用 `recycle.Move` 移入仓库回收站；前端 `showSummaryDialog` 显示去重统计

### 2. 仓库树右键菜单 → 启用/禁用

- **现状：** ✅ **已实现** — `frontend/js/ui/cm-tree.js` `showFileMenu()` 新增 `⛔ 禁用`/`✅ 启用` 菜单项（根据 `isBannedEntry()` 判断），调用 `window.go.main.App.ToggleModelEnable(path)` 后自动刷新树和整合包状态

### 3. 回收仓库模型后，整合包同步状态联动

- **现状：** ✅ **已实现（方案 B）** — 回收操作不会自动扫描整合包，但底部状态栏的 "▶️ 同步状态" 按钮（`btn-sync-toggle`）点击即可刷新所有整合包状态。
- **备选方案 A（未实施）：** 回收前扫描所有整合包 → 弹出提示 → 推荐同步
