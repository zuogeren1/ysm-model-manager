# v1.0.8 — 2D 模型预览 + YSMParser 集成

> 发布日期: 2026-06-08

## 🎯 核心改动

### 🔗 YSMParser 集成（加密 .ysm 解码）

- 集成 [YSMParser](https://github.com/OpenYSM/YSMParser) CLI 作为 sidecar，解码加密 .ysm 模型
- `findYSMParser()` 自动查找 exe 同目录 / 工作目录 / PATH
- `runYSMParserOnFile()` 调 YSMParser 解密 → 解析 `models/*.json` → 提取骨骼 + 纹理
- 支持解密后 `minecraft:geometry` 格式的完整解析（BlockBench UV 对象兼容）
- YSMParser.exe 需放置于程序同目录，发版时一并打包

### 🖼️ 2D 模型骨骼渲染

- 新增 Canvas 2D 骨骼线条图渲染（前视图 + 俯视图小窗），展示模型骨骼结构
- `renderModel2D()` 自动居中缩放，支持纹理贴图加载
- 颜色可见度修复：填充色 rgba 0.15→0.45，边框 0.6→0.85
- 支持 `.zip`/`.7z`（标准 `minecraft:geometry`）+ `.ysm`（经 YSMParser 解密）

### 🔧 解析兼容性修复

| 问题 | 原因 | 修复 |
|------|------|------|
| YSM 加密模型解析失败 | `texture_width:512.0` 是 float64，struct 声明 int | 改为 `float64`，赋值时 `int()` |
| UV 格式不兼容 | YSM 输出 BlockBench 格式 UV 对象 `{...}` 非数组 `[...]` | Cube.UV 改为 `json.RawMessage` |
| 空骨骼崩溃 | Go nil slice 序列化为 `null`，前端 `for...of` 报错 | `make([]Cube2D, 0)` 保证 `[]` |
| `bone.cubes is not iterable` | 前端迭代缺 null 检查 | 所有 `bone.cubes` 加 `\|\| []` |
| `.zip` 内 YSM 模型 | zip 无 `minecraft:geometry` 时未尝试 YSMParser | zip/7z 失败后 fallback 到 YSMParser |

### 🐛 Bug 修复

- 前端即时占位符：点击模型立刻显示 `🏗️ 模型结构（读取中...）`
- YSMParser 中文路径兼容（Go UTF-16 文件系统访问）
- `wails build -clean` 不丢失 YSMParser.exe（`build-release.ps1` 新增 sidecar 复制步骤）

## 🧪 技术说明

- `.ysm` 是 YSM 专有二进制格式（YSGP header + AES 加密），非标准 zip
- 旧版 YSM 模组的 `UvUnion` 用 `@JsonAdapter` 同时支持数组/对象两种 UV 格式
- YSMParser 输出文件结构：`output/{name}/models/{main,arm}.json` + `textures/` + `animations/`
