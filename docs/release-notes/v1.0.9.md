# v1.0.9 — WASM 内嵌解码 + 模型预览增强

> 发布日期: 2026-06-08

## 🎯 核心改动

### 🧠 WASM 内嵌 YSMParser 解码（P6-1）

- 将 YSMParser 编译为 WASM，base64 内嵌于前端，**彻底消除 exe sidecar 依赖**
- 内存解析路径 `ysm_decode_from_memory()` 直接传入字节数组，绕过 MEMFS 文件 I/O
- 回退路径保留 `callMain`（兼容旧版 WASM 编译）+ Go CLI `AnalyzeBedrockModel`
- 新增 `ysm-wasm-bridge.cpp` C++ 桥接层，暴露 `ysm_decode_from_memory` / `ysm_detect_version` / `ysm_diag_header`
- 编译参数：`-sMODULARIZE=1 -sEXPORT_NAME=YSMParserModule -sINVOKE_RUN=0 -sEXIT_RUNTIME=0`
- 胶水代码注入 `Module["HEAPU8"]=HEAPU8` 解决新 Emscripten 闭包变量访问问题
- 解码来源标识：标题旁显示 `🧠 WASM` 或 `⚙️ CLI` 徽标

### 🗃️ 双解码去重 + 缓存（P6-2）

- 新增 `_previewDataCache = new Map()` 统一缓存纹理 + 几何体
- `_loadPreviewImage` 和 `_loadModel2D` 共享缓存，避免重复解码
- Go 端 `AnalyzeBedrockModel` 一次性返回 texture + geometry

### 🖱️ 骨骼交互高亮（P6-3）

- `calcBoneHitZones()` 计算骨骼屏幕坐标热区
- `mousemove` 悬停 → 亮橙色高亮 + 骨骼名变金色
- `mouseleave` → 恢复正常
- `_hoverCleanup` 防止重复绑定

### 📊 模型统计面板（P6-5）

- 骨骼图上方新增统计标签行：`🦴 骨骼 N · 📦 立方体 N · 📐 纹理 W×H · 📁 格式`

### 🏷️ 解码来源标识（P6-7）

- 加密 .ysm 模型显示 `🧠 WASM` 徽标
- 回退 CLI 路径显示 `⚙️ CLI` 徽标

## 🔧 Build 改进

- YSMParser.exe **不再打包进发版 zip**（WASM 已内嵌）
- `build-release.ps1` 移除 #3 sidecar 复制步骤
- 发版归档不含额外 exe

## 📋 Debug 记录

8 轮调试详记于 `docs/postmortem-20250608-wasm.md`：
1. MEMFS callMain I/O 不可靠 → 内存解析 bridge
2. 新增 C++ bridge 暴露内存解析接口
3. HEAPU8 闭包变量 → 胶水代码注入 patch
4. `const` 重赋值 → `let`
5. `.replace()` 歧义匹配函数定义 → 带分号前缀
6. ALLOW_MEMORY_GROWTH 后 HEAPU8 分离 → malloc 后重取
7. `new Uint8Array(string)` 在 WebView2 返回空 → `Uint8Array.from()`
8. `[]byte` 经 JSON 序列化为 base64 → `atob()` 解码
