# v1.5.8 — 创意工坊修复 & 自动扫描优化 🔧

> 发布日期: 2026-06-14

---

## 🐛 修复

### 组件 404 — 动态导入变量导致 Vite 无法解析路径（严重）

**root cause**: `const _loadWC = (p) => import(p)` 使用变量作为路径参数，Vite/esbuild 构建时无法静态分析变量中的路径字符串。5 个 Web Components（app-tree/app-sidebar/app-content/app-resource-manager/app-sync-manager）全部被跳过，构建结果仅 25 modules（正常应为 98 modules）。

感谢 @zuogeren1 提交 PR #3 定位并修复此问题。

### 创意工坊模型列表不显示

- `#ws-repo-list` → `#gh-repo-list`（2 处容器 ID 不匹配，模型列表无法挂载到 DOM）
- `tryFetchModels` 数据加载静默失败（空数组抛错阻断后续仓库）
- checkbox className 被覆写导致复选框样式异常

### 仓库路径按钮误导

仓库文件树底部 `📁 C:\Users\...` 路径按钮，原行为为打开文件夹，改为**跳转到设置页**以便修改路径。

## ✨ 改进

### 自动扫描全面优化

| 改进           | 说明                                                      |
| -------------- | --------------------------------------------------------- |
| 全盘自动扫描   | C-Z 盘所有可用磁盘，不再硬编码 C/D/E                      |
| 增加启动器支持 | MultiMC / PolyMC                                          |

- `.btn-sm` 补上 `color:var(--txt)`（浅色主题下文字不可见）

- 删除设置页路径值的冗余 `📂` 伪元素图标（避免与标题图标重复）

### 社区索引修复

- 社区索引 URL 从独立仓库 `ysm-creator-index`（404）改为主仓库 `ysm-model-manager` 的 `creators.json`
- 🌐 按钮现在同时拉取 `creators.json` + `workshop_sites.json`（增量合并）
- 新增 `fetchCommunitySites` + `mergeCommunitySites` 函数

### 默认路径恢复

- YSM 模型路径未设置时恢复基于 mcRoot 的默认路径推算（`{mcRoot}\\config\\yes_steve_model\\custom`）
