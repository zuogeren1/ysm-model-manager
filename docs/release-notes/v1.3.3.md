# v1.3.3 — 纹理精度修复 + 旧格式兼容（无 ysm.json ZIP / YSGP V2）

> 发布日期: 2026-06-10

## 🎯 核心改动

### 🖼️ 纹理精度修复（PNG 实际尺寸覆盖几何体元数据）

几何体 JSON 的 `texture_width/height` 可能声明过小（如 32×32），但实际 PNG 为 2048×2048。旧代码只在 `texWidth === 0` 时用 PNG 尺寸覆盖，32 ≠ 0 所以跳过，导致 UV 归一化错误——纹理显示为极小一块。

**修复**：`_decodeYsmViaWasm()` 中读取 PNG IHDR 获取实际尺寸，当 PNG 尺寸 > 几何体元数据时覆盖：

| 层级                | 旧逻辑                                            | 新逻辑                                                  |
| ------------------- | ------------------------------------------------- | ------------------------------------------------------- |
| 骨骼级 `_texWidth`  | `parsed.texWidth > 0 ? parsed.texWidth : maxTexW` | `maxTexW > parsed.texWidth ? maxTexW : parsed.texWidth` |
| 几何体级 `texWidth` | `geometry.texWidth === 0 && maxTexW > 0`          | `maxTexW > geometry.texWidth`                           |

### 📦 无 `ysm.json` 的 ZIP 支持

部分 ZIP 模型包不含 `ysm.json`，只有 `main.json` + `arm.json` + 纹理（如 [就叫纸板] 円香 2023-04）。`ExtractYsmSummary()` 找不到 `ysm.json` 直接报错，前端 `_showModelDetail` 得到空摘要 + 空头部后 throw `"无法解析此文件"`。

**修复**：`go/ysm/summary.go` 的 `ExtractYsmSummary` 中新增降级扫描分支：

- 遍历 ZIP 内文件
- 检测含 `"minecraft:geometry"` 的 JSON → 计为模型文件
- 检测含 `"animation"`/`"controller"` 的 JSON → 计为动画
- 统计 `.png`/`.jpg`/`.jpeg` → 计为纹理
- 用文件名（去后缀）当展示名
- 返回成功（非错误），正常进入渲染流程

### 🔐 YSGP V2 早期模型兼容（BOM+文本头部变体）

2023 年早期模型使用 YSGP V2 格式变体：文件以 **UTF-8 BOM** + `"YSGP"` + 文本头部（Metadata/Tips/Export/SHA-256 节）开头，之后才是加密二进制数据。WASM C++ 解析器无法识别此变体：

| 模型                         | 旧行为            | 原因                           |
| ---------------------------- | ----------------- | ------------------------------ |
| 芙兰朵露 Flandre 2025-06.ysm | ⚠️ 未找到几何数据 | WASM 在字节 0 没找到 YSGP 魔数 |
| 古明地觉 Satori 2025-10.ysm  | 主结构识别为箭形  | 密钥位置偏移导致部分解密错位   |

**修复**（三处联动）：

1. **前端** `buildStdYsgpFromTextVariant()` — 从文本头部提取 `<hash>` 密钥（32 字符 MD5），定位加密数据起始点，拼接为标准 `"YSGP"` + `version(4B, V2)` + `hash(16B)` + `加密数据` 格式，再传给 WASM 解析器
2. **Go** `detectYSGPHeader()` — 支持跳过 BOM 检测 YSGP 魔数，从文本头部提取模型名称
3. **Go** `isYSGP()` — 跳过 BOM 再检查 YSGP 魔数，确保 `ExtractYsmSummary` 对 YSGP 文件走正确分支

### 🐞 其他修复

| 问题                                     | 根因                             | 修复                               |
| ---------------------------------------- | -------------------------------- | ---------------------------------- |
| `AnalyzeYSMHeader` 对 YSGP V2 返回空     | `bufio.Scanner` 读二进制文件失败 | 新增 `detectYSGPHeader` 先检测魔数 |
