> 发布日期: 2026-06-13

## 架构变动

### 新增

| 系统                             | 层级 | 文件                        | 说明                                  |
| -------------------------------- | ---- | --------------------------- | ------------------------------------- |
| `InstallDir` 递归安装            | Go   | `go/installer/installer.go` | 整目录安装，支持子目录递归 + 类型过滤 |
| `CountInstanceResources`         | Go   | `app_install.go`            | 统计整合包可清空的文件数，先计后删    |
| `PullSingleResourceFromInstance` | Go   | `app_install.go`            | 单文件从整合包拉取到全局仓库          |
| `Sidebar 勾选同步`               | 前端 | `app-sidebar/index.js`      | 全选 checkbox + 同步所选按钮          |
| `sidebar:rtype-changed` 事件     | 前端 | 跨组件                      | sync-manager 切换类型时通知 sidebar   |

### 修改

| 系统             | 改动                                         | 说明                             |
| ---------------- | -------------------------------------------- | -------------------------------- |
| **事件系统**     | `hdr.onclick`/`oncontextmenu` → 事件委托     | outerHTML 后事件不丢失           |
| **卡片更新**     | `_updateCardStats` 索引匹配 → name 匹配      | 排序变化时不错位                 |
| **卡片重排**     | `_updateCardStats` 后 `appendChild` 重排 DOM | 切换类型后卡片视觉排序更新       |
| **类型过滤**     | `extMatch` + 文件树 `typeExts`               | 各类型只显示自己的扩展名         |
| **扩展名**       | `.json` 仅 `ysm.json`，`.vrm` 加入 VRC       | 减少干扰项                       |
| **导入路由**     | 按扩展名自动路由到对应仓库                   | 不再全部塞入 YSM 仓库            |
| **文件夹推送**   | MMD/YSM 整文件夹 + 递归子目录                | 纹理/动作文件不遗漏              |
| **禁用状态**     | `SyncStatusDisabled` 类型 + 前端 ⛔ 标记     | .ban/.disabled 文件可筛选        |
| **配置文件修复** | `resource_types.json` 结构重写               | YSM 条目损坏 + resourcepack 重复 |

### 移除/废弃

| 项目                                                     | 替代                                 |
| -------------------------------------------------------- | ------------------------------------ |
| `sidebar-import-all` 一键安装全部按钮                    | 全选 checkbox + 同步所选             |
| sync-manager 搜索框 + 状态下拉菜单                       | 状态标签行（`📊全部` `✅已同步` 等） |

| `instance:clear` handler 中旧 `ClearCustomDir`           | 新 `ClearInstanceResources`          |

## 关键数据流

### 类型联动

```
sync-manager 点「🎭 MMD (30)」
  → _selectedType = "mmd-skin"
  → localStorage.setItem("repo_rtype", "mmd-skin")
  → bus.emit("sidebar:rtype-changed", { rtype: "mmd-skin" })
  → sidebar._updateCardStats("mmd-skin")
    → loadInstances("mmd-skin")  →  GetResourceInstanceStatus
    → 匹配卡片 name → outerHTML 替换头部 → 按 newData 重排 DOM
    → bindFooter + bindInstanceActions
```

### 文件夹推送

```
推送 [vup]子言-水手服.../xxx.pmx
  → InstallResourceToInstance(rtype="mmd-skin", srcPath=.../xxx.pmx, ...)
  → 检测 srcParent != globalRoot → needsFolder=true
  → installer.InstallDir(srcParent, dstDir, globalRoot, linkMode, rtype)
    → 创建 dstDir/[vup]子言-水手服.../
    → 遍历 srcDir 文件
      → isAllowed: MMD 只允许 .pmx/.pmd/.png/.tga/.spa/.sph
      → 过滤掉 .vrm 文件
    → 递归子目录（spa/textures/toon/...）
    → 每文件 linkOrCopy / CopyFile
```

### 清空操作

```
右键「清空此整合包的模型」
  → bus.emit("instance:clear")
  → CountInstanceResources(insName)  → 遍历所有 6 个子目录
    → countInstanceDir → filepath.WalkDir 递归统计
  → 0 文件 → toast 提示，跳过
  → modalConfirm 显示文件数 → 确认
  → ClearInstanceResources(insName)
    → clearInstanceDir → filepath.WalkDir 递归收集 → 删除 → 清理空目录
```

### 勾选同步

```
勾选整合包 → 点「同步所选」
  → 收集勾选包的 name
  → 逐包 emit sync:download-missing({ instanceName, rtype })
  → handler 调 GetResourceInstanceStatus 获取缺失 → InstallModelTo
  → 每包完成后 emit sync:download-complete → 解 Promise
  → 所有包完成后按钮恢复
```

## 测试要点

1. **类型切换**：sync-manager 点不同类型标签 → sidebar 卡片数字/排序/按钮文字是否同步刷新
2. **文件夹推送**：MMD 文件夹推送后检查 `3d-skin/EntityPlayer/` 下文件夹结构完整、纹理文件存在
3. **勾选同步**：只勾选部分整合包 → 点同步所选 → 是否只处理勾选的
4. **清空预览**：右键清空 → 先显示文件数 → 确认后检查文件被删除
5. **扩展名隔离**：VRC 标签下不出现 `.pmx` 文件；YSM 标签下不出现非 `ysm.json` 的 `.json` 文件
6. **导入路由**：拖入 `.pmx` 文件 → 检查是否导入到 MMD 仓库而非 YSM 仓库
7. **禁用状态**：开关 `.ban` 文件 → 切换后按钮立即刷新
