## ✨ 改进

### 整合包"打开文件夹"按类型导航子目录

右键整合包 → "📂 打开文件夹" 现在会根据资源类型自动导航到正确的子目录：

| 资源类型    | 打开目录                       |
| ----------- | ------------------------------ |
| YSM 模型    | `config/yes_steve_model/custom/` |
| 材质包      | `resourcepacks/`               |
| 光影包      | `shaderpacks/`                 |
| 蓝图        | `schematics/`                  |
| MMD 皮肤    | `3d-skin/EntityPlayer/`        |
| VRChat 模型 | `vrchat-avatars/`              |

**智能回退**：如果标准子目录不存在，自动扫描版本目录下所有子目录找包含该类型文件的目录，都找不到则回退到实例根目录。

### 子目录映射统一管理

- `SubDirAll()` — Go 端统一维护子目录映射表
- `GetSubDirMap()` — 新增 Wails Binding 暴露给前端
- 右键菜单"打开文件夹"和"复制模型清单"均从此获取映射，**一处修改处处同步**

### 右键菜单显示资源类型

右键标题行现在显示 `rtype`，如 `📦 我的整合包 (create-blueprint)`，方便确认当前类型。

## 🐛 修复

### `GenerateRepoIndex` 循环外 append（严重）

通义灵码审计发现 `GenerateRepoIndex` 将 `append` 放在循环外部，使用了错误的变量。已修正。

### `OpenFolder` 缺少目录存在性检查

通义灵码审计发现 `OpenFolder` 在目录不存在时仍调用 explorer 会报错，已增加 `os.Stat` 前置检查。

### `OpenInstanceFolder` 缺少空值检查

新增 `instDir == ""` 的防御性检查，避免空路径调用 explorer。

### `ScanModelEntries` 哈希性能优化

非 YSM 类型大文件（MMD/VRC 可达数十 MB）跳过全量 SHA256 计算，避免扫描卡顿。
