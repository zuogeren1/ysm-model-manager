- **无**：API 破坏性变更。

## 架构决策记录

### 1. 创作者卡片排名算法

- 金银铜牌根据本地仓库 `ScanModelEntries` 统计的模型数实时计算
- 每次打开创作者频道重新计算，不持久化
- 前三名用 `sort.Slice` 取前 3，第四及以后用默认排序

### 2. 头像提取策略

- 仅解析 `.ysm` 格式文件（跳过 `.zip`/`.7z`，避免解压大文件）
- 头像目录：`{mcRoot}/model/avatar/{creatorId}.png`
- 首次解析完后 base64 编码传前端缓存，后续直接从文件读
- 调试入口：点击头像触发 `__debugAvatar` 模式

### 3. 社区索引三路回退

```
GitHub raw → jsdelivr CDN → GitHub API → 本地缓存
```

- 先尝试 `raw.githubusercontent.com`
- 404 后自动切换到 `cdn.jsdelivr.net`
- 再失败走 `api.github.com`
- 全部失败静默使用本地缓存，不弹 Toast

### 4. 数据安全

- 所有写入先写 `.tmp` 文件，成功后 `os.Rename` 原子替换
- 编辑模式下删除/排序/新增前收集所有打开的输入框
- 恢复机制：如果启动时检测到 `.tmp` 残留，尝试恢复

### 5. 外链统一

- 所有 `window.open` 替换为 `OpenInBrowser`（Wails `Browser.OpenURL`）
- 消除浏览器弹窗拦截、共享登录态、统一行为
