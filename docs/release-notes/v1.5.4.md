# v1.5.4 — 发布修复 & 稳定性补丁

> 发布日期: 2026-06-13

---

## 🐛 修复

### 发布包遗漏 resource_types.json

EXE 运行时报"暂无资源文件"、查重失效、仓库读取不了 MMD/VRC。根因：`build-release.ps1` 未将 `resource_types.json` 复制到发布目录，Go 端 `os.ReadFile("resource_types.json")` 找不到注册表文件，所有依赖资源类型的函数返回空。

同步管理器、`DetectResourceType`、`LoadResourceTypes` 等函数均受影响。

### MMD/VRC 仓库扫描卡死

切换到 🎭MMD 或 🥽VRC 子标签时一直显示"⏳ 扫描中..."。根因：`ScanModelEntries` 对所有匹配扩展名的文件调用 `computeFileHash()` 读取全文件算 SHA256。MMD 的 `.pmx`/`.pmd`（10-50MB）和 VRC 的 `.vrca`（可达百 MB）导致扫描卡死。

修复：仅对同步系统需要的 YSM 类型（`.ysm`/`.zip`/`.7z`/`.json`）计算哈希，其他类型跳过。

### 同步管理器类型标签不更新侧栏卡片数字

点击 💎YSM / 🎭MMD / 🥽VRC 标签后，侧栏的 📦 数字始终显示 YSM 计数。根因：类型切换事件只覆盖了全局子标签入口，同步管理器漏发了 `repo:rtype-changed`。

### Wails v2.12.0 兼容

`wails.json` 移除已废弃的 `"build": { "dir": "build" }` 属性，适配新版 CLI。
