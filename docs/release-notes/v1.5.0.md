## ✨ 新特性

### 文件树虚拟滚动

- 几千个文件时 DOM 节点不再爆炸（固定行高 28px，仅渲染可视区域 + 上下缓存 15 行）
- 文件夹行和文件行拍平为一维数组，`padding-top/bottom` 撑出滚动高度
- 事件委托代替逐元素绑定，虚拟滚动换 innerHTML 后事件仍然有效
- 新增 `virtual-scroll.js` 模块（`ROW_H=28`、`BUFFER=15`、`requestAnimationFrame` 节流）

### 键盘快捷键

- `Ctrl+F` / `Cmd+F`：聚焦搜索框并全选文本
- `Delete`：删除当前选中的文件（与右键菜单「移入回收站」功能一致）

### 资源库 ↔ 整合包对接（Phase 1-3）

**Phase 1：单文件推送**
- 新增 `InstallResourceToInstance` Go Binding — 将任意资源文件复制到指定整合包的对应子目录
- 资源管理器「📦 推送」按钮 → 弹窗选整合包 → 复制文件（已移除，统一走右键菜单）
- 新增 `modalSelect()` 通用下拉选择弹窗

**Phase 2：整合包资源标签页**
- 整合包详情页由「模型列表 + 资源标签」改为统一标签行：
  - 🧱 YSM / 🎭 MMD / 🥽 VRC → `app-tree` 文件树 + 搜索
  - 🎨 材质包 / ☀️ 光影包 / ⚙️ 蓝图 → `app-resource-manager` 扁平列表
- `app-tree` 支持通过 `root` 属性加载不同资源类型（YSM/MMD/VRC）
- `app-resource-manager` 新增 `instance` 属性支持，路径通过 `ListVersionInstances` 按实例名推导

**Phase 3：双向同步**
- 新增 `SyncResources()` — 文件名级比对全局目录与整合包目录，返回 `{synced[], missing[], extra[]}`
- 整合包详情页顶部同步状态条：⬆️ 待推送 / ⬇️ 可拉取 / 🔄 已同步
- 新 Binding：`PushResourceToInstance` / `PullResourceFromInstance`，批量推送/拉取未同步文件

### 安装器扩展

- `installer.Install()` 扩展名白名单增加 `.pmx` / `.pmd` / `.vrca` / `.nbt` / `.schematic`
- `InstallResourceToInstance` 改为调用 `installer.Install`，复用硬链接/符号链接模式
- 文件树右键菜单新增「📦 推送到整合包…」— 所有模型类型统一入口
- 移除 `app-resource-manager` 的独立 📦 按钮（统一走右键菜单）

### 整合包页面统一

- 「通用资源库」页面（📚）合并入「模型仓库」（📦→📚），导航栏移除重复入口
- 仓库页顶部操作栏 + 第二栏资源类型标签双栏布局
- 旧「通用资源库」书签自动重定向到模型仓库

### 工具栏精简

- 📥 导入 / 🗑️ 删除 / 📁 打开 / 🔄 刷新 合并为「⋮ 更多」下拉菜单
- 现在只有：搜索框 + 🎨 作者 ▾ + ⚡ 批量 ▾ + ⋮ 更多 ▾ + 排序
- 回收站图标统一为 ♻️（♻️ 移入回收站 / ♻️ 清空回收站 / ♻️ 标签页）

## 🐛 Bug 修复

### 选中高亮消失
- 根因：`selectState.keys` 存绝对路径，`flattenVisible` 查相对路径，永远不匹配
- 修复：`entryKey = e.fullPath || e.path`，与 `data-fullpath` 保持一致

### 选中后 stat 被覆盖
- 根因：`updateStat` 发现文本不是"共 N 项…"格式后进入动画分支，700ms 后覆盖
- 修复：有选中项时 `_renderTree` 跳过 `updateStat`；`oldTotal=0` 时不走动画

### 右键无 Ctrl 新增选中
- 根因：右键处理器无条件把右键文件加入 `selectState.keys`
- 修复：仅在已有选中项时才追加

### CSS 功能缺失
- 旧 `app-tree.css`（74 行）含 `.fh .ar.open` 旋转动画、`.fl.flash` 闪烁反馈，未迁移到 Shadow DOM 版
- 修复：补充两项缺失 CSS 规则
- 删除孤儿文件 `frontend/js/components/app-tree.css`

### 其他修复
- `updateStat` 数字跳动动画兼容非标准文本
- `reload()` 未传 rtype 导致非 YSM 资源类型刷新后跳到默认仓库
- 清理 `events.js` 无用的 `flashBtn` 导入
- 移除 `toolbar-events.js` 中已不存在的按钮 handler（`btn-dedup`/`btn-trash`/`btn-pv`）
- 右键菜单移除「打开所在文件夹」（经验证，仓库级打开体验更好）
- 批量右键菜单移除「全部启用」「全部禁用」（文件夹无法批量选中）

## 🔧 重构

- **文件树事件系统**：从逐元素 `querySelectorAll().forEach(el.addEventListener)` 改为容器级事件委托
- **CSS 通用化**：删除重复 `.hdr-btn` / `.repo-srch` 定义，删除 orphaned `app-tree.css`
- **死代码清理**：移除未使用的 `_pushToInstance` 方法、`_initResources` 页面、`resourceLibraryHTML`
- **`push-to-instance` 统一**：资源推送从 `copyFileTo`（自建）改为 `installer.Install`（复用链接模式）
