## 🎨 UI 改进总览

本次更新对前端 UI 系统进行了全面优化，涵盖无障碍支持、响应式布局、代码规范、性能优化等多个方面。

### 核心改进

#### 1. **无障碍支持（WCAG 2.1 合规）** ✅
- 添加 `:focus-visible` 焦点状态
- 键盘导航时显示清晰的焦点环
- 鼠标点击不显示焦点（避免干扰）
- 符合 WCAG 2.1 无障碍标准

**影响文件：**
- `frontend/css/variables.css` - 焦点状态样式

---

#### 2. **统一按钮样式系统** ✅
- 创建 `.btn-base` 基类
- 三种尺寸变体：`.sm`（工具栏）、默认、`.lg`（主要操作）
- 语义变体：`.primary`、`.danger`、`.accent`
- 保留现有按钮类（向后兼容）

**新增变量：**
```css
--btn-radius: 6px;
--btn-padding-sm/md/lg: 三种尺寸
--btn-transition: 过渡动画
```

**影响文件：**
- `frontend/css/variables.css` - 按钮变量
- `frontend/css/components.css` - `.btn-base` 基类

---

#### 3. **CSS 变量语义化重命名** ✅
- `--free` → `--status-success`（成功/免费状态）
- `--paid` → `--status-error`（错误/付费状态）
- `--sz-green` → `--size-ok`（文件大小正常）
- `--sz-red` → `--size-large`（文件大小过大）
- 添加向后兼容别名（@deprecated 标记）

**影响文件：**
- `frontend/css/variables.css` - 变量定义 + 别名

---

#### 4. **响应式布局支持** ✅
- **>1200px**：三栏布局（侧栏 + 主内容 + 预览）
- **900px - 1200px**：两栏布局（隐藏预览面板）
- **<900px**：单栏布局（仅主内容）
- 平滑过渡动画（0.18s）

**影响文件：**
- `frontend/css/layout.css` - 媒体查询

---

#### 5. **性能优化** ✅
- 移除 `.gh-card` 常驻 `will-change: transform`
- 优化过渡动画：`transition: all` → 指定具体属性
- 减少不必要的图层保留，降低内存占用

**影响文件：**
- `frontend/js/components/app-content/content-css.js` - 移除 will-change
- `frontend/css/variables.css` - 优化 transition

---

#### 6. **代码质量提升** ✅
- 移除 `!important` 滥用（4 处）
  - `.dp-log-fbtn.active`
  - `.cr-local-btn:hover`
- 提高样式可维护性和优先级可控性

**影响文件：**
- `frontend/js/components/app-preview/preview-css.js`
- `frontend/js/components/app-content/community-site-view.js`

---

#### 7. **Z-Index 层级系统** ✅
- 定义 7 个语义化层级变量
- 替换 14 处硬编码 z-index
- 统一管理，避免魔法数字

**层级定义：**
```css
--z-dropdown: 100        /* 下拉菜单 */
--z-sticky: 200          /* 吸顶元素 */
--z-modal-backdrop: 900  /* 模态框背景 */
--z-modal: 1000          /* 模态框内容 */
--z-popover: 1100        /* Popover/Tooltip */
--z-toast: 1200          /* Toast 通知 */
--z-fullscreen: 9999     /* 全屏预览/拖拽覆盖层 */
```

**影响文件：**
- `frontend/css/variables.css` - 层级变量定义
- 11 个 CSS/JS 文件 - 替换硬编码 z-index

---

#### 8. **代码清理与文档化** ✅
- 删除重复的设置页样式（13 行）
- 重命名 `scrollbar.css` → `transitions.css`（更准确）
- 添加 CSS 变量系统文档化注释（22 行）

**影响文件：**
- `frontend/js/components/app-content/content-css.js` - 删除重复
- `frontend/style.css` - 更新引用
- `frontend/css/variables.css` - 文档化注释

## 🔧 技术细节

### 向后兼容性

所有改动都保持了向后兼容：
- 旧按钮类仍然可用（`.tb-btn`, `.dlg-btn` 等）
- CSS 变量别名映射（`--free` → `var(--status-success)`）
- 无破坏性改动

### 性能影响

- ✅ 移除常驻 `will-change` 减少内存占用
- ✅ 指定 transition 属性避免不必要的 layout/paint
- ✅ 响应式布局使用 CSS Grid，性能良好

### 可维护性提升

- ✅ 统一的按钮系统，新代码可直接使用
- ✅ Z-Index 层级系统，避免层级冲突
- ✅ CSS 变量文档化，降低学习成本
