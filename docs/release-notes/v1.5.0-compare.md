## 关键数据流

### 推送链路（统一）

```
右键文件 → context-menus.js → InstallModelTo / InstallResourceToInstance
                               → installer.Install(src, dstDir, root, linkMode)
                               → linkOrCopy / symlinkOrCopy / CopyFile
```

### 同步链路

```
SyncResources(globalDir, instanceDir)
  → filepath.Walk 两个目录
  → 文件名级比对 → ResourceSyncResult{synced[], missing[], extra[]}

PushResourceToInstance / PullResourceFromInstance
  → SyncResources → 遍历 missing/extra → installer.Install / copyFile
```

### 文件树渲染（虚拟滚动）

```
_renderTree()
  → buildTree(entries) → 嵌套对象树
  → flattenVisible(node, depth) → 一维 row 数组 [{type, key, html, depth}]
  → calcVisibleRange(container, totalRows) → {startIdx, endIdx}
  → 渲染 visible rows + padding-top/bottom 撑高度
  → installScrollSync (rAF 节流)
```

## 测试要点

1. **虚拟滚动**：切换文件夹展开/折叠时滚动位置是否跳动；搜索过滤后 flat list 是否正确重建
2. **资源推送**：MMD/VRC 推送到整合包后目标子目录是否正确；硬链接模式是否生效
3. **同步检测**：`SyncResources` 在有 .disabled 后缀时的文件名匹配
4. **事件委托**：虚拟滚动换 innerHTML 后点击/右键/开关是否正常
5. **键盘快捷键**：Delete 在焦点不在树内时是否误触发；Ctrl+F 聚焦后 Escape 是否能恢复
