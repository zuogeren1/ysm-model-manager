# v1.4.3 — YSM 头部解析修复 + 同步状态修复

> 发布日期: 2026-06-11

## 🔧 Bug 修复

### YSM 文件头解析彻底修复

- **二进制数据泄露**：`scanHeader` 在文本头部结束后继续读入二进制数据，导致 UI 显示乱码。新增退出条件：遇到段落后 `---` 分隔线即停止扫描。
- **纯二进制 YSGP 文件**：部分加密 YSGP V2 文件没有文本头部，`scanHeader` 不应被调用。新增 `hasTextHeader()` 检测，只有包含 `--- [`、`<name>` 等文本标记的文件才进行头部扫描。
- **`<free>` 标签不存在时显示付费标记**：新增 `HasFree` 字段区分「未标注」和「明确付费」。只有文件头包含 `<free>` 标签时才显示 🆓/🔒 标记。

### 整合包同步状态修复

- **哈希去重导致文件误标为已同步**：`repoByHash` 从 `map[string]types.ModelEntry` 改为 `map[string][]types.ModelEntry`，同哈希的多个文件全部保留在 Missing 中，不再误标为「已同步」。

### 预设搜索按钮 HTML 修复

- 修复 `data-q` 属性缺少闭合引号导致的 HTML 结构破坏，按钮现在有正确文字标签。
