# v1.6.4 — FilesRoot 统一存储架构 + 设置页 UI 翻新 + 下载修复

## 变更内容

### FilesRoot 统一存储架构

PR #6（zuogeren1）的架构变更：

- `RepoRoot` 字段 → `FilesRoot` + `YsmRoot`，所有资源统一存 `{FilesRoot}/{子目录}/`
- `GetRepoRoot` 重写：专属覆写 → FilesRoot + StorageSubDir 两层回退
- 新增 `StorageSubDir()` 统一子目录映射
- 新增 `specificRoot()` 查专属覆写路径
- 配置文件自动迁移：旧 `repoRoot` → 新 `filesRoot` + `ysm` 子目录

### ZIP 内容检测路由

- Go 端新增 `DetectZipType` Binding，扫描 ZIP local file header
- `.zip` / `.7z` 调 Go 检测内容：pack.mcmeta → 资源包，shaders/ → 光影包，ysm.json/models/ → YSM
- 前端 DnD/导入队列全面支持 `shouldEnterForm()` 异步检测
- 非 YSM 文件（.pmx/.vrca/.nbt）直接导入跳过命名表单

### 蓝图整合 .litematic

- Go `extensions.go` + 前端 `extensions.js`/`icon.js` + `resource_types.json` 三端同步
- `.litematic` 合并入 `create-blueprint` 类型

### 设置页 UI 翻新

- 三栏卡片布局：游戏根目录 + 链接模式 + 下载镜像源
- 文件存储路径独立行，可展开详细调整面板
- 链接模式/下载镜像源从散落 flex 改为标准 stg-card
- 精简下载镜像描述文字
- 鸣谢独立 tab（🎯 灵感来源 + 🙏 特别鸣谢）
- 版本卡片化，`dev` 版本号放标题栏右侧
- `.stg-page` padding 改为内联样式修复 Shadow DOM 隔离
- 设置页滚动修复

### 下载修复

- Go `download:progress` 参数顺序修复：`(total, downloaded)` → `(downloaded, total)`
- 心跳去掉 `total > 0` 条件，未知大小文件也能正常更新进度
- 前端 `total <= 0` 时不再秒跳 100%
- `download-queue.js` 添加 `GetRepoRoot` import

### 整合包预设移除

- 删除 `data.js` 中 `fallbackInstances()` 的 3 处调用
- 无整合包时返回空数组，不再显示假数据

### Go 端配置迁移

- `AppConfig` 加回 `RepoRoot` 字段（用于 unmarshal 旧配置）
- `loadAppConfig()` 检测 `FilesRoot==""` 且 `RepoRoot!=""` 时自动迁移并写回

## 影响范围

- **重大**：配置字段 `repoRoot` → `filesRoot`（自动迁移，用户无感）。前端 26 处路径从硬编码 `(cfg.filesRoot||"")+"\ysm"` 改为 `GetRepoRoot("ysm")`。
- **轻微**：设置页布局变化、下载进度参数修正。
- **无**：API/接口变更。
